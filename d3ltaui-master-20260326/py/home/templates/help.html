User Manual

This guide explains how to use the D3lta UI to upload a CSV, launch processing with D3lta, follow task progress, and download analysis results.

1) Before you start

  • Prepare a non-empty .csv file with a header row.
  • Make sure at least one column contains natural-language text (sentences, messages, posts, etc.), because this is the column you will analyze.
  • Keep your Task ID after upload. You will need it to check status and retrieve results.

2) Upload a CSV and configure processing

  1. Open Upload CSV in the top menu.
  2. Set processing parameters (or keep defaults):
  • Threshold Grapheme (0-1): D3lta threshold_grapheme.
  • Threshold Language (0-1): D3lta threshold_language.
  • Threshold Semantic (0-1): D3lta threshold_semantic.
  • Minimum Text Size: D3lta min_size_txt (positive integer).
  • Truncate Text To: maximum characters used per cell from the selected text column. Values above the configured platform maximum are capped automatically.
  • Use random N rows: optional random sample size. Use 0 (or empty) to process the entire CSV.
  1. Choose your CSV file in CSV File.
  2. Wait for the automatic preview. The UI reads your file and shows:
    • Detected columns
    • Line count
    • A preview table (first rows)
  3. In Text Column Name, select the column containing text to analyze. This field is required.
  4. Optional: choose Column Separator manually (Auto, comma, semicolon, tab, pipe) if auto-detection is not correct for your file.
  5. Click Submit.

After submission, the form is locked and an upload progress bar is displayed. When upload succeeds, you are redirected automatically to the Task Status page.

3) Keep and reuse your Task ID

  • On the status page, use the Copy button next to the Task ID while status is PENDING or PROCESSING.
  • You can later open Task Status from the menu and paste the Task ID in the search form.

4) Understand task statuses

  • PENDING: file accepted; waiting for processing.
  • PROCESSING: analysis is running.
  • DONE: results are ready to preview and download.
  • FAILED: processing failed; details may be available in Error Details.
  • EXPIRED: result retention time passed; re-upload is required.

For pending/processing tasks, the status page refreshes automatically about every 30 seconds when the status changes.

5) Review and download results (DONE tasks)

When a task is DONE, the page shows two result areas.

Matches

  • Displays a preview of the first 10 rows.
  • Use the Download button to download the full matches CSV (with item count and file size shown).

Clusters

  • Select a cluster from the dropdown (All Clusters, Non-clustered, or a specific cluster).
  • The preview loads dynamically for the selected cluster.
  • Use the corresponding Download button to export full cluster data.

6) Troubleshooting

  • Upload failed / invalid input: verify all required form fields, especially Text Column Name, and check numeric parameters are valid.
  • Task not found: confirm the Task ID was copied fully and without extra characters.
  • Processing failed: retry with default parameters and ensure the selected column contains natural-language text (not IDs, dates, or numeric-only values).
  • Expired: results are no longer available; upload the CSV again to create a new task.

7) Notes and best practices

  • Start with defaults, then tune thresholds if needed.
  • For large datasets, use Use random N rows to run quick trial analyses first.
  • If some text cells are very long, set Truncate Text To to balance speed and detail.
